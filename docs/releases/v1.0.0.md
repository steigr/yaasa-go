# Release v1.0.0 plan and execution

Date: 2026-05-19

## 1) Define release scope and version

Status: done

- Release version is fixed to `v1.0.0`.
- Scope for this release:
  - Stable FE60 control CLI (`cmd/yaasa`)
  - Stable local daemon IPC behavior (`internal/ipc`)
  - Stable FE60 packet parser behavior (`internal/protocol`)
- Out of scope for `v1.0.0`:
  - New protocol opcode support beyond currently documented behavior
  - Platform expansion beyond currently verified build matrix

## 2) Stabilize code and verify quality

Status: done

Executed checks:

```bash
go test ./...
go test -race ./...
```

Result: all package tests passed, including race detector.

## 3) Validate API/CLI/protocol compatibility

Status: done

Validation done:

- CLI command surface validated with:

```bash
go run ./cmd/yaasa --help
```

- Protocol behavior baseline documented in `docs/protocol.md` and covered by tests in `internal/protocol/packet_test.go`.
- Added explicit version command (`yaasa version`) so released binaries can be traced to source commit and build timestamp.

## 4) Prepare artifacts and docs

Status: done

Prepared docs:

- `README.md`
- `CHANGELOG.md`
- this release execution document

Prepared release build flow:

- Build metadata injection via `Makefile` `LDFLAGS`
- New targets:
  - `make release-check`
  - `make release-artifacts VERSION=v1.0.0`

Artifact build commands used:

```bash
make build build-linux build-mac-arm
```

Known caveat:

- `make build-mac-intel` may fail on Apple Silicon hosts without an amd64-compatible CoreBluetooth/cgo cross-compile setup.
